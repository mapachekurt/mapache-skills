# Agent2Agent (A2A) Protocol Specification (v1.0 Summary)

## Core Objects

### 1. Task
The unit of action in A2A.
- `id`: Unique task identifier.
- `status`: Lifecycle state (Submitted, Working, Completed, etc.).
- `artifacts`: Output generated by the task.
- `history`: Message history associated with the task.

### 2. Message
A single unit of communication.
- `role`: USER or AGENT.
- `parts`: Container for content (Text, File, or Data).

### 3. Part
- `text`: Natural language content.
- `file`: Reference to a URI or bytes.
- `data`: Structured JSON blob.

### 4. Artifact
The formal output of a task.
- `artifactId`: Unique reference.
- `parts`: The content of the artifact.

## Agent Discovery
Agents describe themselves using an **Agent Card**:
- `capabilities`: (Streaming, Push, Extensions).
- `supportedInterfaces`: (JSON-RPC, gRPC, HTTP+JSON).
- `skills`: List of distinct capabilities.
